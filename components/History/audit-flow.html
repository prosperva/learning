// Audit Log System

Flow Breakdown

  • 1
    Middleware Sets Identity
    ResourceResolutionMiddleware runs on every request — populates ICurrentRequestResource.ResourceId from JWT claims before the controller even runs.
  • 2
    EF Detects Changes
    When any repository calls SaveChangesAsync(), EF Core's ChangeTracker has all Modified / Added / Deleted entities queued up.
  • 3
    Interceptor Fires
    SavingChangesAsync intercepts BEFORE the SQL executes. Entity states are still Modified / Added / Deleted — perfect for capturing what changed.
  • 4
    IAuditableEntity Filter
    Only entities implementing IAuditableEntity get audited. Add the interface to any entity you want tracked — zero other changes needed.
  • 5
    Separate DbContext Write
    Audit logs are written via CodesAuditLogDbContext — a completely separate context. Prevents infinite loops and keeps concerns isolated.
